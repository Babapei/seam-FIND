# 焊缝寻位方法详细对比表

---

## 一、方法总览

| 方法 | 类型 | 输入 | 输出 | 是否需要训练 | 训练数据量（若要训练） |
|------|------|------|------|--------------|------------------------|
| **2D 流派**（depth_valley / depth_edge） | 传统几何 | 深度图 (+ 可选彩图) | 焊缝 3D 轨迹 | **否** | — |
| **截面法 + PCA** | 传统几何 | 3D 点云 | 焊缝 3D 轨迹 | **否** | — |
| **PointNet** | 深度学习 | 3D 点云 | 缝点 / 3D 轨迹 | **是** | 约 500～5000+ 帧（视场景复杂度） |
| **PointNet++** | 深度学习 | 3D 点云 | 缝点 / 3D 轨迹 | **是** | 约 1000～10000+ 帧 |
| **DGCNN** | 深度学习 | 3D 点云 | 缝点 / 3D 轨迹 | **是** | 约 1000～10000+ 帧 |

---

## 二、各方法具体内容

### 2D 流派（depth_valley / depth_edge）

| 项目 | 内容 |
|------|------|
| **原理** | 深度图是一张「图」，每像素存距离。焊缝在深度上多为凹槽（局部深度更小）。在 2D 上找缝线，再用相机内参把 (u, v, depth) 反投影到 3D。 |
| **典型做法** | depth_valley：按行/列扫描，每行在局部窗口内找深度最小的像素 → 2D 缝线 → 反投影；depth_edge：边缘检测。 |
| **输入** | 深度图 (H×W) + 相机内参 (fx, fy, cx, cy)；可选彩图（用于 image_edge）。 |
| **输出** | 焊缝 3D 轨迹 (N×3) |
| **优点** | 不训练、计算快、易调试、与结构光「先有图后有点云」一致 |
| **缺点** | 依赖深度质量，噪声会带来轨迹抖动 |
| **训练** | 无需训练 |
| **标签** | 不需要 |

### 截面法 + PCA

| 项目 | 内容 |
|------|------|
| **原理** | 沿焊缝大致走向切若干截面，在每个截面内对点云做 PCA 找主方向/中心，把这些截面中心连成 3D 轨迹。 |
| **典型做法** | 先估计焊缝走向 → 按法向切截面 → 截面内 PCA 找主成分或中心点 → 排序得到轨迹。 |
| **输入** | 3D 点云 |
| **输出** | 焊缝 3D 轨迹 |
| **优点** | 不训练、纯几何、适合规则焊缝 |
| **缺点** | 对复杂、弯曲焊缝需额外处理 |
| **训练** | 无需训练 |
| **标签** | 不需要 |

### PointNet

| 项目 | 内容 |
|------|------|
| **原理** | 每个点经共享 MLP 得特征，再用 max-pooling 聚合成全局特征，结合点特征做逐点分类（缝/背景），缝点排序后得到轨迹。 |
| **典型做法** | 点云 → 共享 MLP → max-pooling → 点特征+全局特征 → MLP 输出每点二类 logits。 |
| **输入** | 点云 (N×3)，通常 N=2048 或 4096 |
| **输出** | 每点类别 logits → 缝点 → 排序得到 3D 轨迹 |
| **优点** | 对无序点云、置换不变；结构简单 |
| **缺点** | 局部几何信息较弱，主要靠全局特征 |
| **训练** | 需要 |
| **标签** | 需要逐点二类标签（缝/背景） |
| **参数量** | 约 3～4M（sem_seg 结构） |

### PointNet++

| 项目 | 内容 |
|------|------|
| **原理** | 分层结构：先局部采样+分组，在局部做小 PointNet 提取特征，再逐层抽象，最终得到每点特征做分割。 |
| **典型做法** | 最远点采样选中心 → 球/距离内取邻域 → 局部 PointNet → 特征传播到全点。 |
| **输入** | 点云 (N×3) |
| **输出** | 每点类别 → 缝点 → 3D 轨迹 |
| **优点** | 局部几何更好，对小结构更敏感 |
| **缺点** | 实现复杂，需编译 tf_ops（原版 TF） |
| **训练** | 需要 |
| **标签** | 需要逐点二类标签 |
| **参数量** | 约 1～2M（通常比 PointNet 略少但结构更复杂） |

### DGCNN

| 项目 | 内容 |
|------|------|
| **原理** | 用 K 近邻建图，在图上做边卷积（EdgeConv）：对每条边提取特征，再聚合更新点特征；逐层更新图。 |
| **典型做法** | 建 K-NN 图 → EdgeConv（边 = 两点的差 + 一点坐标）→ 多层更新 → 得到点特征做分类/分割。 |
| **输入** | 点云 (N×3) |
| **输出** | 每点类别或全局分类 |
| **优点** | 显式建模邻域关系，局部几何强 |
| **缺点** | 计算比 PointNet 重，图构建有开销 |
| **训练** | 需要 |
| **标签** | 需要逐点二类标签（分割时） |
| **参数量** | 约 1～3M |

## 三、核心区别对照

| 对比维度 | 2D 流派 | 截面法+PCA | PointNet | PointNet++ | DGCNN |
|----------|---------|------------|----------|------------|-------|
| **输入形式** | 深度图（2D） | 点云（3D） | 点云（3D） | 点云（3D） | 点云（3D） |
| **处理域** | 2D 图像域 | 3D 点云域 | 3D 点云域 | 3D 点云域 | 3D 点云域 |
| **是否深度学习** | 否 | 否 | 是 | 是 | 是 |
| **是否需要训练** | 否 | 否 | 是 | 是 | 是 |
| **是否需要标签** | 否 | 否 | 是 | 是 | 是 |
| **局部几何建模** | 有（局部窗口） | 有（截面内） | 弱 | 强 | 强 |
| **对噪声敏感度** | 中等（深度噪声） | 中等 | 中等 | 较低 | 较低 |
| **实现难度** | 低 | 中 | 低～中 | 高（编译） | 中 |

## 四、2D 流派 vs 截面法+PCA

| 项目 | 2D 流派 | 截面法 + PCA |
|------|---------|--------------|
| **输入** | 深度图 | 点云 |
| **思路** | 先在 2D 上找缝 → 再反投影到 3D | 直接在 3D 点云上按截面找缝 |
| **是否同一方法** | 否，算法流程不同 | |
| **共同点** | 均为传统几何方法，均无需训练 | |

## 五、训练数据量参考（深度学习方法）

| 方法 | 最少建议量 | 较推荐量 | 说明 |
|------|------------|----------|------|
| **PointNet** | 约 500 帧 | 2000～5000 帧 | 结构简单，数据需求相对少；焊缝场景单一可偏少，多样则需更多 |
| **PointNet++** | 约 1000 帧 | 3000～10000 帧 | 分层结构更复杂，需要更多样本才能充分学习 |
| **DGCNN** | 约 1000 帧 | 3000～10000 帧 | 图结构+邻域建模，数据不足易过拟合 |

**说明**：
- 「帧」= 一帧点云 = 一个独立样本（如一次扫描/一帧深度图反投影得到的点云）
- 焊缝形态、视角、材质越多样，越需要更多数据
- 若有仿真/合成数据做预训练或混合训练，可适当减少真实标注量
- 以上为经验区间，实际需根据验证集表现调整

## 六、一帧点云的含义

| 概念 | 含义 |
|------|------|
| **一帧** | 一次采集/一次扫描得到的一整团点云，可视为一个独立样本 |
| **训练时** | 例如「1000 帧」= 1000 次不同的扫描，每次对应一张点云图（可能包含几千到几万个点） |
| **PointNet 输入** | 通常对每帧采样固定点数（如 2048 点）送入网络 |
| **与图像类比** | 一帧点云 ≈ 一张图像；1000 帧点云 ≈ 1000 张图像 |
